Unit 3: Program Design and Debugging

Converting Algorithms to JavaScript

Finally - we get to write real code! For this course, we use JavaScript, but all of the design work we've done so far in Unit 3 could be used with any programming language. Our task is to translate our inputs, outputs, and algorithm into JavaScript.

Algorithm to JavaScript

Here's how we translate our algorithm verbs into JavaScript code.

Getting Input From Users

Algorithm statements like: read, get, prompt that ask for strings that a user must key in (like first name). In JavaScript, the code would look like this:


var name = "";
name = prompt("Enter a name:");

This code opens a dialog box with a message that will say: "Enter a name:". The user then keys in a name and clicks the OK button. JavaScript will take the value keyed in by the user and store that value in the variable called: "name". Note that we defined the variable before we used it with the var statement. We also gave it an initial value. While we're not required to do that in JavaScript, it's a Very Good Idea.

Handling Numeric Input

Algorithm statements like get number value, prompt for numbers, accept as integers, anything meant to ask the user to key in numbers (like age) need to be handled differently:


var age = 0;
age = prompt("Enter your age in years:");
age = Number(age);  // Convert the entered string to a number!

This is a really important part of translation from algorithm to JavaScript. When you enter something to program from a prompt dialog box - JavaScript sees it as a string. Even if you enter "27" it is still a string composed of the letters "2" and "7". But - that could cause a BIG problem, because we can't do addition with strings in JavaScript!

So, we have to tell JavaScript to convert the strings that we entered into numbers. To do this we use the convenient function "Number()". You put a string, or a variable containing a string, in between the parentheses and it is converted to a number.

The code line above age = Number(age); is a perfect example! When the user keys in the age, even if it is a "22", JavaScript thinks it is a string. So the "Number()" function turns it back into a number.

Everything that comes from the keyboard is a string. If you want to do math on data you got from the keyboard, convert it to numbers.

Handling Output

Algorithm statements like: print, output, display mean that we have to display something on the screen. We have a few options in JavaScript:


document.write("Hello, " + name + "!");

We've used this already. This will add what's inside the parentheses to the HTML page.

We can also use alert boxes:


alert("Hello, " + name + "!");

Although this is more annoying to users because an alert box pops up on the screen and the user has to make it go away.

Finally, we can inject text into an existing region of the page. Say we have a div element on the page like this:

<div id="message">
</div>

We can use JavaScript to inject text into that region:


var messageArea = document.getElementById("message");
messageArea.innerHTML("Hello, " + name + "!");

We use document.getElementById() to grab a page element with JavaScript. We then use the innerHTML property to set the new text that goes inside of that element. We won't use this approach in this class, but this is the preferred method for working with HTML and JavaScript; we use HTML to structure the content and use JavaScript to alter it. This is called DOM manipulation.

You won't use DOM manipulation in these learning units. You will use document.write and prompt as they will keep the exercises simple and will resemble patterns you'll use in other programming languages that don't interact with HTML. For example, in Java you'll use System.out.println() instead of document.write() to display things to the screen.

However, for additional practice, I encourage you to to rewrite the programs using DOM manipulation. This will allow you to make more professional-looking programs.

Processing Data

Algorithm statements like: compute, average, summarize, accumulate, calculate, increment, decrement, math operators (add, subtract, divide, multiply) + and - (plus and minus) are asking us to do computations. We do those in JavaScript code like this:


total = number1 + number2 + number3;
counter = counter + 1;
averageAge = (age1 + age2 + age3 + age4)/4;

These math operators in JavaScript are the same as in English! But, the trick in JavaScript is that the name of the variable that will hold the answer must be on the LEFT side of the equal sign!

The examples above are also called "assignment statements" because we are assigning a value to each of the variables. For example, if number1 = 5 and number2 = 4 and number3 = 3, then JavaScript will assign a value of 12 to the variable called: total.

Writing Our JavaScript Code

As you recall, our algorithm in Pseudocode looks like this...

AddThreeNumbers
  Initialize number1 to 0
  Initialize number2 to 0
  Initialize number3 to 0
  Initialize output to 0
  Initialize total to 0

  Prompt for number1 with "Enter the first number"
  Prompt for number2 with "Enter the second number"
  Prompt for number3 with "Enter the third number"

  Convert number1 to a number
  Convert number2 to a number
  Convert number3 to a number

  Set total = number1 + number2 + number3

  output =  "The total of " + number1 + " + " +
          number2 + " + " + number3" +
          " is " + total + "."

  Display output
END

...and so we translate it into JavaScript like this:



// Add three numbers

// Declare the variables
var number1 = 0;
var number2 = 0;
var number3 = 0;
var output = ""
var total = 0;

// Get the numbers from the user
number1 = prompt("Enter the first number:");
number2 = prompt("Enter the second number:");
number3 = prompt("Enter the third number:");

// Convert the inputed strings to numbers
number1 = Number(number1);
number2 = Number(number2);
number3 = Number(number3);

// Calculate the total
total = number1 + number2 + number3;

// Display the total. You can split the line of code at a + sign
output = "The total of " + number1 + " + " + number2 + " + " +
               number3 + " is " + total + ".";

document.write(output);

That's some good looking code! Do you get the impression that there will often be more JavaScript code than pseudcode? You're right!
