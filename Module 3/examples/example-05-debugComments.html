Example 5: Using Comments to Help Debug JavaScript Code

File: /examples/example-05-debugComments.html

Source Code:

Here is some JavaScript code that is not producing any output. We know that it should because that is what the document.write statement is supposed to do! 


// Here is JavaScript code that doesn't work - but why?
var month = 5;
var day =  10;

document.write("Here is the month " + month);
document.write("<br>);
document.write("Here is the day " + day);

Demo Output:

Using Comments

Look, there's no output. Maybe you can spot the error right away. But what if you can't? One way to spot the line that has the error is to look at the code line by line. If we "hide" one line from JavaScript, and then get output, then the line that we hid must be the one with the error! And how can we "hide" code from JavaScript? COMMENTS!!

A comment is a handy tool to use to isolate problems in code. We can comment lines of code to block them from JavaScript. Then we can remove the comments one line at a time until we find that bad line! Sometimes debuggers aren't enough to help us find the problem and this is a tried and true technique in every programming language out there.

Let's try commenting out the last document.write command. Then if we save and refresh the browser - is there any difference?

Source Code:


// Here is JavaScript code that doesn't work - but why?
var month = 5;
var day =  10;

document.write("Here is the month " + month);
document.write("<br>);
//    document.write("Here is the day " + day);

Demo Output:

Shucks...no difference. Okay - try commenting out the second to the last document.write command. Save and refresh the browser. Is there any difference now?

Source Code:


// Here is JavaScript code that doesn't work - but why?
var month = 5;
var day =  10;
    
document.write("Here is the month " + month);
//    document.write("<br>);
//    document.write("Here is the day " + day);

Demo Output:

Explanation:

When you commented out the second to the last document.write command you suddenly got output!. That indicated that there was quite possibly an error on that line. Yup, looks like we forgot the closing quote in the break tag. Now we will fix that error, remove the comments and try again! It's called DEBUGGING!

Repaired Source Code:


// Here is JavaScript code after debugging!
    var month = 5;
    var day =  10;
    
    document.write("Here is the month " + month);
    document.write("<br>");
    document.write("Here is the day " + day);
