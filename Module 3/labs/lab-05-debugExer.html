Lab 5: Debugging Exercise

File: /labs/lab-05-debugExer.html

Did you notice that there is no output below in the "Demo Output" box? There are several common errors in the code below - and those errors are preventing our output from displaying correctly. Follow these steps and use the Web Console and comments to practice your debugging skills, and eventually get all of your output to display correctly.

The error console won't catch everything. But, it will catch some errors - and lead you to places near other errors. Once you have exhausted the help the error console offers, it's time to start using comments to systematically hide lines of code from JavaScript until you can identify the line that has an error.

Debugging instructions:

  1. While this page is open in Firefox, open the Web Console (select Firefox, Web Developer,,Web Console).
    • Let's make sure you are starting "fresh"...with no errors leftover from previous windows. Click Clear.
    • Close the Web Console window.
    • Now refresh your browser, and open the Web Console window again.
  2. Look at the first error message on the Web Console.
    • The error message is pointing to the line that reads: var test2 10;
    • Notice the line number.... We can locate that line number in Notepad ++. We don't need to do this now, but sometimes it helps to click the link to have the Error Console show you the exact line with the error it has identified.
    • Click the Clear button and close the Web Console.
    • Proceed to Notepad ++, fix the error, save the file, and refresh your browser! (missing equal sign)
  3. Still no output! There must be more errors. Open the Web Console again.
    • The error message is pointing to the line that reads: document.write(this is a string);
    • Click the Clear button and close the Web Console.
    • Proceed to Notepad ++, fix the error, save the file, and refresh your browser! (missing quotes around the string)
  4. Ok, we still have some problems, so what's the next error? Let the console tell you.
    • Looks like it has something to do with another document.write. See the problem?
    • Fix up the error and refresh the page. Any progress?
  5. Still no output! Good grief - there must be more errors. Open the Web Console again.
    • This time the error message is pointing to a line that includes a break tag. What's missing in that syntax?
    • Note the line number, then click the Clear button and close the Web Console.
    • Proceed to Notepad ++, find that line, fix the error, save the file, and refresh your browser! (missing quotes)
  6. And.... nothing again! Open the Web Console again and see what's up.
    • Another break tag?? If people just wouldn't use these we wouldn't have to debug them!
    • Note the line number, then click the Clear button and close the Web Console.
    • Proceed to Notepad ++, find that line, fix the error, save the file, and refresh your browser to see if it worked.
  7. Again...no output! Will this never end?!?!? Open the Web Console again.
    • The error message is pointing to the line that includes the "my favorite thing" line. Looks like another syntax issue.
    • Click the Clear button and close the Web Console.
    • Proceed to Notepad ++, fix the error, save the file, and refresh your browser! (double quotes around the string might do it)
  8. No output again. Open the Web Console again.
    • The error message sys "document is not defined". What's that all about???
    • Note the line number. Can you spot what's wrong? Click the Clear button and close the Error Console.
    • Proceed to Notepad ++, fix the error, save the file, and refresh your browser! (oops, it's a spelling error!)
  9. At last - we have output!. But, we don't have all of our output. Open the Web Console again.
    • Same error message as last time - but a different line number.
    • Locate that line number in Notepad ++. This time, there just doesn't seem to be an error in this line of code. But, click the Clear button and close the Web Console.
    • We can't fix the error, but we can HIDE that line of code from JavaScript to see if other errors are revealed. So, place the comment characters (two forward slashes) at the very beginning of that line of code, save the file, and refresh your browser.
  10. Let's see if the Web Console will tell us anything else. Open the Web Console again.
    • Look at this error message very carefully!
    • Note that line number, click the Clear button and close the Error Console.
    • Fix that spelling error in Notepad ++, save the file, and refresh your browser.
  11. Again, we have output, but we don't have all of our output. Open the Error Console again.
    • Locate that line number in Notepad ++. It's another situation where there just doesn't seem to be an error in this line of code. But, click the Clear button and close the Web Console.
    • We can't fix the error, but we can HIDE that line of code from JavaScript, too. That will give us another chance to see if other errors are revealed. So, place the comment characters (two forward slashes) at the very beginning of that line of code, save the file, and refresh your browser.
  12. Same situation again. We have output, but we don't have all of our output. And, it's pointing to a line that doesn't seem to have an error. Open the Web Console again.
    • Now we are getting suspicious. Perhaps the line in error is NOT the line number being identified by the Web Console. Click the Clear button and close the Web Console.
    • Let's look at the line just BEFORE the lines we have commented out. AHA!!! Do you see the problem with the "document.write" line that is supposed to display "This is fun"??? Fix those syntax errors, save the file, and refresh your browser.
  13. That seemed to make a difference, but we still should have more output! Open the Web Console again.
    • Ok...this time it's pointing to the "alert" command. Click the Clear button and close the Web Console.
    • Fix the spelling error, save the file, and refresh your browser.
  14. Now that's progress! The "alert" worked...and we now have two lines of output! But we still should have more output. Open the Web Console again.
    • This time, the Web Console shows no errors. Click the Clear button and close the Web Console.
    • Now we'll have to continue by using comments to identify lines with errors that prevent our output from displaying.
    • Somewhere between the line that displays "this is a string" and the alert - there must be an error! Let's "comment out" the line immediately following the line that successfully displays the "this is a string" message.
    • Comment that line in Notepad++, save the file, and refresh your browser.
  15. That helped! We now have an additional line of output! That should make us think that there could be a problem in the line of code we just commented.
    • Take a close look at that line we commented out. The break tag is missing something! Could that tiny error have really caused all of this trouble? YES!
    • Remove the comment characters from that line in Notepad++, fix the syntax error, save the file, and refresh your browser.
  16. Even better! We now have our message box working, along with three lines of output with working break tags! But, we're still missing output...
    • Let's try removing the comment characters from the other two lines we commented out in the debugging process. We didn't really see any errors in those lines, anyway.
    • Remove the comment characters, save the file, and refresh your browser.
  17. Almost perfect! We now have everything but one line of output displaying.
    • The only line that doesn't display is the second from the last "document.write" statement. If you look carefully, you'll see that there is, indeed, a syntax error.
    • Fix the syntax error (which character is used for concatenation???), save the file, and refresh your browser.
  18. OH JOY - It worked!!! We now have the message box and all of our lines of output displaying.
    • We successfully debugged our program line by line, using both the Web Console and comments.
