Variables in JavaScript

Computer programs mostly deal with data. In this section we're going to look at what data is and how we use a VARIABLE to store it and manipulate it.

Computers

Computer programs do three basic things from our perspective.

  • They get data into the program
  • They process the data
  • They output the data

So What's Data?

Let's first look inside the computer at RAM. RAM stands for Random Access Memory. This means that the computer can go get something in RAM from anywhere very quickly. RAM is where the computer stores its data, including the operating system and the programs themselves. How do we imagine RAM? Here's one favorite way:

RAM is one long linear chain of bits. And a bit is a very simple thing, it's just a 0 or a 1. Think of each mark on the tape measure as a single bit. Data is just a collection of bits in a known spot. Say, starting at 5" on the tape measure and going for 10 marks.

What's a Variable?

A computer program has to have a way to find and use the data that it needs. It does this with variables. A Variable is a spot in RAM that has been given a name. You can think of variables as a container. But, this container doesn't hold water, it holds bits.

Here's how we do that in JavaScript:


// Variables in JavaScript

var count;
  • We start with the word var. This tells JavaScript that you are about to create a named variable. It is required by JavaScript in order to correctly make a variable.
    • Omitting the var keyword causes the variable to be declared in what's called the "global scope." Without going into too much detail now, just know that it's bad, it wastes memory, and can slow down some web browsers. Just don't do it. Always use the var keyword.
  • Then we have the name of the variable, in this case count.
  • And, finally, we have a semicolon (;), which is required at the end of simple statements like this, per our coding standards.

However, at this point the variable doesn't have any value in it. Its value is "null." Here's the variable declaration over again with a small addition.


// A variable with an initial value

var count = 1;
  • We've added the = 1 (a space, an equal sign, and the number 1).
  • This line creates the variable and puts the value 1 into it. This is called an assignment statement.

Declaring a variable and assigning it a value at the same time is called "initializing" a variable. And you saw that terminology in your pseudocode: 

  Initialize count to 1

Displaying output

Before we can dive into the labs, we need to be able to see what is going on in our programs.

Here's a piece of JavaScript that you've seen already. We'll explain it in more detail now.


// Our first way to display output on a web page

document.write("This is a string");
  • The word document refers to the HTML page that contains this JavaScript program. It's actually a reference to an object called the DOM, or Document Object Model which is a tree of elements on a web page. HTML tags create elements on the DOM, and we can use the document object to manipulate that tree.
  • The period (.), called the dot, is a way to access things inside of your document.
  • The word write is a function that causes text to be displayed on a web page. It actually modifies the DOM.
  • Next is a set of parentheses with some quoted words inside. The words inside the quotes are the words that get displayed. The quotes don't get displayed though.
  • And, finally, a semicolon to end the command...sort of like the way a period ends a sentence.

What happens when this is run by JavaScript?

    • When this function is run it takes whatever is in the parentheses and outputs it to the HTML page right in the same location where the "write" code happens to be. In our labs that will be just after the opening <script> tag ends.

What can I put in a "write"?

  • Everything that we are creating in our programs can go in there. It's the most simple way to output things from our program, and is very similar to constructs in other languages.

Data Types

How a computer keeps its data straight

All data in a computer is really the same, it's all just zeros and ones. However, your programs will need to think of different pieces of data as different types of data. For example, you can't add the word "Bob" to the number 1000 mathematically. They are two different TYPES of data values! All programming languages have a set of "data types". Here's JavaScript's:

  • Numbers
  • Strings
  • Booleans
  • Null
  • Undefined
  • Objects

We will focus on the first three for most of the semester. This is a very simple list, some languages have a much bigger list. JavaScript is simpler in this area. Let's look at more details about the data types

Numbers

Some of the most important things we do with computers involve numbers.

The number data type in JavaScript holds, well, numbers. Numbers are things we can do math operations on. We can count with them, have them be prices, add them together, divide them, etc. These are all numbers:

4
$1.50
-14
1000000
  • Numbers can be big or small, positive or negative, have decimals or not. In JavaScript they are all just numbers. There aren't different data types for different kinds of numbers.

How do we make a variable that's a number?

  • That's easy, just declare a variable and put a number in it. Just make sure you type the number with no quotes around it.

// Numbers in JavaScript

var count = 1;
var price = 12.99;
var priceString = "12.99"; // This is not a number!
  • We don't have to tell JavaScript that the variable is a number? Nope.
    • That's not true in some other languages like C# and Java though. JavaScript is a dynamically-typed language, and the data type is controlled by the variable. JavaScript programs use type inference to determine the data type. That means the program looks at the data and figures out its type. In other languages we have to explicitly declare the type when we code the program.

Strings

We often have to process strings of text - not just numbers - in programs too.

Strings are a series of characters surrounded by double or single quotes.


"This is a string"
"This isn't a number"
'This is a string too'
'The cow says, "moo"'
"Fred"
"Bert"
"Ernie"

You must have quotes around a string.

  • If a string is surrounded by double quotes then you can have single quotes inside the string.
  • If a string is surrounded by single quotes then you can have double quotes inside the string.
  • But, sometimes we need to put a character into a string that is special, like a tab. How do we do that?

Escape Characters.

Here's a list of characters that we occasionally need to be able to embed into strings

\b Backspace
\t tab
\n newline
\v vertical tab
\f form feed
\r Carriage return
\ Literal backslash
\" Double Quote
\' Single Quote

To embed these characters in a string you just type the corresponding code with the backslash. Here's an example:


// Embedding special characters

var tabInThisVariable = "This is a tab\tand this is after it";
var aNewLineInTheMiddle = "This line has a newline\nin its middle.";

Concatenation

Concatenation is the process of placing things right next to each other...so close that they are touching. For example, imagine that you placed a brick on your coffee table. You then place a second brick on the coffee table, and slide it right up next to the first brick so that they're touching end-to-end. You could say that you "concatenated" those two bricks!

In programming, we often concatenate pieces of data. We may need to do it just to make our output look nice. (For example, putting first and last names together with a space between them makes names easier for humans to read.) But, sometimes we need to concatenate smaller pieces of data into larger strings to change or clarify the meaning of data, too. (For example, you could concatenate a string like "Salary: " next to a number so that a user understands that the number is a salary.) Or you can put a name with an age:


  var age = 42;
  var name = "Homer";
  document.write(name + " is " + age + " years old.");

Separating processing from output

Concatenation is technically a processing step. You're adding strings together.

Separating processing from output is important. There are better ways to output things to a page using JavaScript, but we'll stick with document.write() for now. But to prepare for later, you may consider turning code like this:.


var age = 42;
var name = "Homer";
document.write(name + " is " + age + " years old.");

into something like this:

var age = 42;
var name = "Homer";
var output = ""

output = name + " is " + age + " years old.";

document.write(output);

In the second version, we use a variable to build up the message. Then we use document.write() to display that message. In small programs it might not matter, but in larger programs it'll matter a great deal, as you'll need to build up an output message over the course of your program. So try to get into the habit of doing outputs separate from string concatenation.

There are lots of routines, or functions, included in JavaScript that let us manipulate strings. Here is one of them.

  • Length of a string: You use the ".length" function to have JavaScript determine the size of a string in bytes. In other words, how many characters are in a specific string? In this code example, JavaScript will count the number of characters in "Homer Simpson", and assign that number to the variable called whatWillThisBe. Note that it even counts the SPACE between the first and last name as a character!

// The length of a string

var output = "";
var whatWillThisBe = "Homer Simpson".length;

output = "The number of characters is " + whatWillThisBe;
document.write(output);
