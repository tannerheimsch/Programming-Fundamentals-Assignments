  • If you DID get output, but it's all on one line...you need some line-breaks (we call them break tags).
  • document.write(); does not make its output go on the next line. You have to do that yourself. Here's how to do it:
      • You could add a blank line like this:
        • document.write("\n");
      • But normally you'd add the characters \n to the end of the string, inside the quotes, like this:
        • document.write("Hello there!\n");
      • That's the newline character, and that's the preferred method in most programming languages.
      • When using JavaScript within HTML pages, newline characters are ignored unless the output is wrapped in <pre> tags.
        • The output sections of each learning unit (where you will type your code) are set up to accept newline characters.
      • The other option, when working with HTML, is to use document.write to insert the <br> tag, like this:
        • document.write("<br>");
    • You will need to use one of the above methods when you want the next piece of data to be on the next line.
      • You will need either three break tags (One after each of the first three document.write commands) or you'll need to put \n at the end of each line of text, within the quotes. Remember, either approach is acceptable, but the \n approach is universal to any program in any language, whereas <br> only works in HTML pages, as the browser processes the <br> character and turns it into a line break.
  • Your output should look exactly like this:
Homer Simpson
123 Some Street
10
12.99
  • So when do you use <br> vs \n?
    • In output that will be displayed on a web page, use <br>. You can even append it to each line of output:
      • document.write("Some text<br>");
    • For other kinds of programs where the text is not being displayed inside of HTML, like JavaScript prompts, alert boxes, etc, use \n.
  • Remember, in our labs, both will work because of how I coded the output section. Be comfortable with both approaches, but be consistent in a program.
