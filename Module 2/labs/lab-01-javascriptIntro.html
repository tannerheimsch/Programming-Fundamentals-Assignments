Lab Steps

  1. Open this file in Firefox and Notepad++.
  2. Add your name.
  3. In Notepad++, go to this spot in the code, within the <script> block:

    // YOUR CODE STARTS AFTER THIS LINE:

    and add a line of text describing your favorite food. The text you key in is NOT JavaScript code, so it will be ignored by JavaScript. You will see the text in your code, but it will not display on the web page.
  4. Save the file.
  5. Refresh the page in Firefox.
  6. Scroll down to the dark gray box labeled: YOUR OUTPUT. Do you see anything? (No? Good!). Remember this...if it's not code that JavaScript recognizes...it just won't show up!
  7. Now add this line to the code:

    document.write("JavaScript is fun.");

    • Just type it, don't copy and paste, it helps you learn.
  8. Save, Refresh, and View
  9. Hey, that line was valid JavaScript code, so why didn't it work?!?!?!
  10. It didn't work because of the "non-JavaScript" code you typed in first. It caused the GOOD JavaScript code to fail! And, sometimes when JavaScript fails, it doesn't tell you why - it just doesn't display any output. That can make it kind of hard to "debug".
  11. Now add two forward slashes to the beginning of your line of "non-JavaScript" code, so it looks like this:
    //the line of text describing your favorite food
  12. Save, Refresh, and View
  13. What happened? You SHOULD see the "JavaScript is fun." text in that gray output box on the web page now!
  14. Sometimes we like to hide lines of code or other text on purpose. We want the code in the HTML file - but we don't want it to display on the web page. We call those hidden lines "comments".
  15. Now add two forward slashes to your line of code, so it looks like this:
    • //document.write("JavaScript is fun.");
  16. Save, Refresh, and View
  17. What happened? You DO NOT see the "JavaScript is fun." text on the web page any more...but we know it's there in the code. We just learned that by using comments, the programmer has control over what displays and what does not!
  18. Now, remove those two slashes (the comment characters)...and save, refresh, and view again. There it is...our output is back.
  19. But, what if we'd made a mistake, a syntax error? Try changing the "d" in the document.write command to an "x"...just to see what happens. Your code should look like this:
    • xocument.write("JavaScript is fun.");
    • Sometimes we make errors...just to see what happens! It's a great way to LEARN!
    • Save, Refresh, and View
    • What happened? The "JavaScript is fun." text is gone! If JavaScript doesn't understand the command, it just ignores it.
  20. Okay, now replace the "x" with a "d" again to fix the syntax error. And then add this line of code after the first one:
    • document.write("<h1>A Big Hello</h1>");
  21. Refresh the page and look for the big hello.
  22. Interesting observation here...the document.write command causes text to display right on the web page, whereas the alert command that we used earlier results in a popup message box. Hmmm...that may be good to know in the future.
  23. All done.
