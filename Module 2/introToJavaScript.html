Introduction to JavaScript

We're using the JavaScript programming language to learn the basics of computer programming. This is a great language to learn with. All you need to use it is a modern Internet browser, like Firefox. You can use simple and free text editors, like Notepad++ or Brackets.io to write JavaScript code.

What is JavaScript?

JavaScript is an interpreted programming language. That means there is a full computer program that reads your JavaScript code and runs each line, one at a time. This program is called an "interpreter." Each line of code gets turned into machine code and executed. Each time you run the program, your code has to be interpreted.

There is a JavaScript interpreter embedded in browsers. You can't really tell that it is separate from the browser itself, but it's there. JavaScript is the language that gives our web applications behavior.

You tell the interpreter what to do with a list of instructions. In this class you'll write JavaScript in the context of HTML pages running inside a web browser.

For JavaScript to work correctly it has to be associated with HTML files correctly. We can attach external JavaScript files, or we can place JavaScript inside of HTML. For now, we are going to place our JavaScript code right inside our HTML documents by a special HTML tag, the <script> tag. Like this:


<script>
// YOUR CODE STARTS AFTER THIS LINE:

// END OF YOUR CODE
</script>
  • Line 1: The <script> tag tells the web browser to treat your code as JavaScript instead of trying to read it as regular text or HTML tags.
  • Line 2-4: You put your JavaScript code in here and it will be handed to the JavaScript interpreter by the browser.
  • Line 2: This line starts with //. That makes the line a comment that will be ignored by JavaScript.

About Rules

In computer programming, there are different levels of rules. Here are the three we are concerned with in this course.

  1. Syntax
    • If a computer programming language enforces a rule then we call that rule "syntax". This is the strongest level of rule. You can't break these rules or your program just won't run.
  2. Convention
    • If virtually all the users of a programming language all over the world do something the exact same way then that way is called a "convention". The JavaScript language itself doesn't care about these rules. However, if everybody is using the same convention there is good reason for it. If you want to join the ranks of professionals using JavaScript then you need to follow these conventions.
    • In this course, JavaScript won't enforce these rules but your instructor will. Part of the points for projects are on conformance to the course coding standards which includes variable names, indentation, editor settings, and spacing of characters. Go here to see those: JavaScript Coding Standards
  3. Style
    • There are some ways of structuring our code where there isn't any agreement as to the best way to do it. A large group of programmers will do it one way and another large group will do it another way. When a rule is in this category we call it a "style". Styles are not enforced by JavaScript or the instructor. What is required is that you pick a style for your code and then be consistent. Make sure that you use that style in all your code. We will go over some common styles in class.

JavaScript Syntax Rules

Remember those rules about computers? They can't think and they can't figure out what you mean. You have to be very clear and you have to use their language, not yours. Programming languages are very simplistic compared to human languages, but they are very strict. You have to follow the rules of the language that are called its syntax. Here are some of the basic syntax rules of JavaScript.

JavaScript is Case Sensitive

In JavaScript, everything has to be the correct case. For example, the word "IF" is not the same as "if". This is a challenge for lots of beginning programmers. We have to start "seeing" the case of text as part of the meaning. Almost everything you learn in programming is case sensitive. And if you mess up the case, you'll often get a syntax error (if you're lucky), or invalid results (if you're unlucky!)

White space are characters you can type on the keyboard but you don't normally see on the screen or on paper when you print. Here are the white space characters:

  • Space
  • Tab
  • Carriage Return (officially called the "line terminator")

In JavaScript, white space usually doesn't matter. Unfortunately, there's that word "usually" in that rule. There are some situations where it matters a lot. Therefore we're are going to have a course standard for using white space. Refer to the course JavaScript Coding Standards for more details.

Comments

Comments in our code are lines of text that are ignored by JavaScript. We use these to write notes to ourselves and other programmers who see our code. This is called Documenting our code.

There are two kinds of comments in JavaScript, single-line and multi-line.

A single-line comment starts with two forward slashes: // Example:

// Here is a single-line comment.
// JavaScript will ignore text that starts with
// the double slashes until the end of the current line.

It's actually OK to put a single-line comment on the same line as some code. Many programmers do this to make it more clear what things do.

var studentGrade;     // The grade for the student stored as a percentage

However, comments like this are often bad practice! The programmer could have just written this with a much more meaningful variable name:

var studentGradeAsPercentage;

Strive to use meaningful names so you don't need to use comments to explain your code.

A multi-line comment starts with /* and ends with */. These can be over many lines. Example:

/* This is a multi-line comment.
It has several lines.
JavaScript will ignore all these lines. */

Sometimes we can use comments to prevent code from running. 

// alert("this line won't run");
alert("this line will");

This is called "commenting out" code and it can be useful when we're testing our software.

Semicolons

Semicolons are used in JavaScript to end a line. Most of the time they are not required by JavaScript. However, they are required in a few places so as a course standard, the use of semicolons at the end of statements is required. You'll find that this is also a convention in the industry when it comes to writing JavaScript code.

Examples:

// Semicolons

var name;
name = "Fred";
return name;

var total = 0;
total = total + 100;

JavaScript Statements

JavaScript statements are lines of text that contain tokens, operators, and identifiers that the JavaScript interpreter understands. They are like the sentences of a programming language. When someone writes prose they write sentences that are organized into paragraphs. When we write JavaScript we write statements that are organized into blocks and functions.

Statements usually end with a semicolon ";"

There are two kinds of statements, simple and compound

Simple statements are one line that just does one thing:

// Simple JavaScript Statements

x = 4;

Compound statements combine multiple levels of logic:


// Compound Statements

if (statusCode === 0) {
  alert("Everything is working fine!");
} else {
  alert("Really bad things happened.");
}

Reserved Words

There are a bunch of words that have special meaning to JavaScript. They are called Reserved Words. Because of this we can't use those words in our code. Here they are:

break delete function return typeof
case do if switch var
catch else in this void
continue finally instanceof throw while
default for new try with
document window prompt alert confirm
name Number in of const let

This is far from a complete list.
