What is an Algorithm?

A process or set of rules to be followed in calculations or other problem-solving operations, esp. by a computer : a basic algorithm for division.

Defining the problem

One way to think of computer programming is breaking down a process into small, manageable, and programmable pieces. Then we make pieces of a program out of each part of the process. Putting all the pieces together into a working computer program is the next step.

Developing algorithms is a part of the process we go through. This learning activity is about developing those algorithms.

An algorithm is a way of doing something. A shopping list of things we need to do. Assembly instructions for a new bike. Steps in dividing two numbers. Starting a car.

Algorithms in computer programming are more strict than assembling a bike. A computer algorithm needs to be followed perfectly, every time. We can't have creativity when we are doing math. Computers may be dumb, but if we correctly tell them to do something they will be very fast and consistent every time we say, "Go!"

The First Breakdown

The first thing we do when breaking down a process is to identify three things:

  1. Input: a list of the source data provided to the problem.
  2. Output: a list of the outputs required.
  3. Processing: a list of actions needed to produce the required outputs.

We start with a problem statement that defines what a program needs to do.

A program is required to read three numbers, add them together and print their total.

The first step is to identify the nouns in the problem statement. These are candidates for inputs and outputs.

A program is required to read three numbers, add them together and print their total.

We can see from this that the inputs are three numbers and the output is the total.

Next, we identify the verbs and adverbs in the statement.

A program is required to read three numbers, add them together and print their total.

These help us determine the processing that the programming needs to do.

Defining Diagram

Once we have identified the inputs, outputs, and processing we make a defining diagram to help us document and visualize the algorithm. While we could do this as a three column table, called an IPO Chart, for simplicity we'll just use plain text. Let's look at an example:

We have three numbers that we want to add together. To make our Defining Diagram, we'll first fill in the inputs and outputs and then add the processing in between.

INPUTS:
  number1
  number2
  number3

PROCESSING:
  read three numbers
  add numbers together
  print total number

OUTPUTS:
  total

It's important to have all processing steps in the Defining Diagram. Make sure to include the input steps, any processing steps, and the output steps.

This is really handy for algorithms with lots of tiny steps we need to work out. But make sure that the processing steps are in the right order!
