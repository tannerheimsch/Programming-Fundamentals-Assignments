Unit 1: Introduction and Program Design

The Structure Theorem

The Structure Theorem helps us break down a problem or a system into small parts that can be easily coded. It is also known as The Structured Program Theorem.

The theorem states that it is possible to write any computer program by using only three basic control structures that are easily represented in pseudocode. Here they are:

  • Sequence
  • Selection
  • Repetition

Sequence

This is putting one foot before the other. You have to tell a computer to do one thing at a time, in a sequence of statements.

  • In pseudocode we will see things like this:
prompt user for name
display name
set firstName to Fred
set lastName to Flintstone
fullName = firstName and lastName with a space in between
add fullName to names list
  • If the order isn't correct in each of those blocks of pseudocode, then things won't work when we convert the pseudocode into programming code.
  • For example, the SEQUENCE of the following statements is very important! You wouldn't get the correct answer for "area" if you didn't have the correct values in "length" and "width" first!

// Statement

length = 4;
width = 5;
area = length * width;

Selection

Selection means that our program needs to make a decision to go one way or another. It does this by first testing something. Is the door locked? Is this number smaller than 500? Is the user logged in? Once it has done the test it will then run one of two statements or sequence of statements.

In pseudocode we use a simple IF, THEN, ELSE block to represent this

IF condition p is true THEN
  statement(s) in true case
ELSE
  statement(s) in false case
ENDIF

Here's a pseudocode example that uses the IF, THEN, ELSE block to show SELECTION:

Check to see what the price is
IF the price is less than $10
  pay with my $10 bill
ELSE
  pay with my charge card
ENDIF
Collect the receipt and go home

In JavaScript code we also use an if construct that looks like this:


// Selection in JavaScript

if (something === 1) {
  // some good code here
} else {
  // some really good code here!
}

Repetition

REPETITION refers to a set of statements or instructions that will be performed over and over. Walking, doing the dishes, and playing tennis are examples of activities where certain steps are repeated over and over. In programming repetition is also called iteration or looping.

In pseudocode we write it as:

WHILE condition is true
  statement block
  another statement block
ENDWHILE

Or

DOWHILE condition is true
  statement block
  another statement block
ENDDO

We sometimes need to do a process until it's done. Like washing dishes:

WHILE there are dirty dishes in the stack
  dip dish in soapy water in sink
  rub dish with sponge
  rinse dish in clean water
  place dish in draining rack to dry
ENDWHILE

Or perhaps we need to hand out pens to everyone until we run out of pens. One way we can express this is like this:

WHILE I have pens in my hand
  give one to a student
END
Tell everyone I'm out of pens.

Each time I give a pen away, I have one less pen in my hand. When I no longer have any pens, I'm done. Note that the steps inside the repeating process are indented. But the last step, where I tell people I'm out of pens, is outside. It happens after I have given all my pens away.

We can use repeating processes to count up to a value. 

  Initialize count to 0
  WHILE the count is less than 10
    Display count
    add 1 to count
  ENDWHILE

And we can even do selection inside of repeating processes.

  Initialize count to 0
  WHILE the count is less than 10
    IF count is an even number THEN
      Display count
    END
    add 1 to count
  ENDWHILE

In JavaScript we will be using a few different repetition structures. Here are two examples.


// Repetition with a while loop

var count = 0;
while(count < 10) {
  document.write(count);  // prints out the count
  count = count + 1;      // adds one to the count
}


// Repetition with a for loop

for(var count = 0 ; count < 10; count++) {
  document.write(count);
}

Don't worry, we won't be writing these kinds of things in JavaScript just yet. This is just a chance for you to see what's coming.
