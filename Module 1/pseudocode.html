Pseudocode

How to Write Pseudocode

Pseudocode is a plain English way to write down algorithms. It uses a bit of structure so that it resembles a programming language, but it is not a programming language. There is no one correct way to write pseudocode, everyone could just do it however they want. But, to make it more useful we will have some guidelines for how to do it this semester.

Six Basic Computer Operations

  • A computer can receive information
  • A computer can assign a value to a variable or memory location
  • A computer can put out information
  • A computer can perform arithmetic
  • A computer can compare values and select one of many alternative actions
  • A computer can repeat a group of actions

A computer can receive information.

The basic verbs we use are things like read, get. or prompt.

prompt for first name

You might also need to get information from the computer, like the system date from the operating system:

get system date

Once we get data, we need to do something with it.

A computer can save things by assigning a value to a variable or memory location.

We create and use "variables" in computer programming and in pseudocode for a few reasons:

  • To give data an initial value
  • To store data we capture from an end user
  • To assign a value as a result of some processing so we can use it later

Variables are just a spot in computer memory (RAM) that we have named. It's a named reference to a spot in the computer's memory, but it's easier for us to remember a name we make up than a hexadecimal number that represents a spot in RAM.

Here are some examples:

initialize totalPrice to zero
set studentCount to 0
totalPrice = costPrice + salesTax

We use initialize or set to give a variable a default value. This is perfect for a variable that we'll use to keep a count of records we process. We need to start at zero and add one to that each time.

It's a really good idea to initialize your values to a default value. Some languages actually require this.

We can also just assign a value to a variable using the = sign. The right side gets processed and the result is assigned, or stored, in that variable.

Here's how we can state that we're getting some information from the user and storing it for later use:

prompt for firstName

Notice we're making it look a little like real code by adding things like using camelCase when there are more than one word (firstName).

However, we really want to make this more clear. How will the user know what to type? We can clarify that in our pseudocode:

prompt for firstName with "Please enter your first name."

You will be expected to be specific with your prompts in your pseudocode assignments.

A computer can put out information.

Computers wouldn't be very useful if they didn't communicate with us. This is called output and it can be as simple as a word or complicated like a video. The verbs are things like put, write, output, display, and print.

display "Program Completed"

This is very simple message, but sometimes the simple ones are really important. For example, we may want to write the output to a file:

write customer record to file

Now, here's one that's a little more vague:

display totalTax

This one is a bit vague. We might not know where we want the output yet. Remember, our pseudocode is a tool that we use to design the program. We can only be as specific as we can with what we know. If we don't know then we'll have to ask more questions and learn more about our problem. Once we found out, we can make it better: 

display "The total tax is $" + totalTax

Note that I'm being very specific, showing that there should be a dollar sign, and then building up some output using the variable.

You will be expected to be specific with your outputs in your pesudocode assignments.

A computer can perform arithmetic.

This is an extremely important function of computers and was one of the many reasons they were developed. Computers can do math and they can do it very fast. They can also be consistent and always give us the same results with the same data. The people who did this before computers were called "computers" because they did computations all day long on an adding machine. The early electronic computers were just faster adding machines. They gave us an advantage during World War II.

add number to total
set total to total plus number
total = total + number

subtract age from currentYear to get birthYear
set birthYear to currentYear minus age
birthYear = currentYear - age

divide total by count to get average
set average to  total divided by count
average = total / count

We can express math functions in an English way or the mathematics way in our pseudocode.

+ add
- subtract
* multiply
/ divide
% modulo (remainders)
() parentheses

The symbols we use in programming for math functions are like the ones you'd use in a spreadsheet. We have to use the asterisk for multiplication and the forward slash (/) for division.

The verbs we use are from math: add, divide, multiply, compute, calculate.

A computer can compare values and select one of many alternative actions

This is the "selection" part of the Structure Theorem and is one of the most important things that a computer can do. It compares the contents of memory, based on some rules, and determines if the contents are equal or not. Most computer languages have the simple word "if" as the operator for this.

Initialize fullTimeCount to 0
Initialize partTimeCount to 0

prompt for studentAttendanceStatus with "Are you a part-time student?"

IF studentAttendanceStatus is "yes" THEN
  add 1 to partTimeCount
  Display "You are enrolled part time."
ELSE
  add 1 to fullTimeCount
  Display "You are enrolled full time."
ENDIF

This is an example of an IF/ELSE statement. If the condition that follows the IF is true, then the code underneath the IF statement is executed. But if the condition is not true, then the code under the ELSE statement is executed.

Notice that the steps beneath the IF statement and the ELSE statement are indented? That shows us visually that those are sub-processes; that they aren't part of the normal flow of our program.

You will be expected to use indentation like this in your pseudocode, and in your program code.

The ELSE statement is optional. Sometimes there's nothing else to do. Sometimes we just need to check something and do an extra step instead of an "either-or" situation. In those cases, we would just do a simple IF statement:

IF user is an administrator THEN
  display "Welcome, administrator."
ENDIF

Each "branch" of a decision statement can have many statements within, which can make programs exponentially harder to maintain and test. And each branch can make further decisions. For example, imagine a program that asks for a state of residence. And then, if you enter "Wisconsin", it will ask you what county you live in. And then if you enter "Eau Claire" for the county, it asks you what city you live in. Sounds complicated, right? Remember that Wisconsin has 72 counties. Now what if you lived in Minnesota or Illinois?

Don't worry, we won't get into that level of complexity just yet. But we will in a few weeks.

Decision processing is an incredibly important part of software development.

A computer can repeat a group of actions.

This is another important feature of computers. Once we have a sequence of tasks to perform, a computer can do those tasks over and over again the same way each time. Remember that computers aren't smart, they're just fast at doing the tasks they have been told to do.

Here's an example program that counts from 1 to 50 and asks for a student's first and last name. It then prints the first and last name out to the screen and adds one to a counter:

Initialize studentTotal to 1
WHILE studentTotal < 50 DO
  Prompt for firstName with "What is your first name?"
  Prompt for lastName with "What is your last name?"
  Display firstName + " " + " lastName + "."
  add 1 to studentTotal
ENDWHILE

The loop completes when the studentTotal is 50, because the condition in the loop says to keep going "while the studentTotal is less than 50.

This is an example of a WHILE loop. It repeats all of the steps between the WHILE and ENDWHILE statements.

Notice that the steps inside of the loop are indented. This makes it easier to see where the steps that will be repeated are.

With the concepts we've explored, we can write pseudocode to accurately describe what our program should do. Sometimes we'll need to make decisions, other times we'll need to repeat some steps over and over. And sometimes we'll even need to repeat some steps and make decisions inside of the part of the code that repeats.

Here's another example of pseudocode:

A program needs to count from 1 to ten. If the number is an even number, it should print out "even". If it's an odd number, then we print out "odd." At the end of the program, it should print "Done".
DisplayOddAndEven
  initialize count to 1
  WHILE count <= 10 DO
    IF (count modulo 2 == 0) THEN
      display "Even"
    ELSE
      display "Odd"
    ENDIF
    set count = count + 1
  ENDWHILE
  display "Done."
END

Notice in this example:

  1. First, we initialize a variable called count to 1.
  2. Next, we create a loop that compares count to 10. If it's less than or equal to 10, we enter the loop.
  3. Next, we check to see if the number is odd. We use modulo to do this. We're checking to see if the number divides evenly by 2 with no remainder. If it does, the number is even. Otherwise it's odd.
  4. Based on the result, we print the appropriate word to the screen.
  5. Then, we increase the count variable by one.
  6. The program will then hit the ENDWHILE line and start the WHILE section again, comparing the count to see if it's less than 10. We started at one, but now it's 2. And so the steps in the loop are repeated.
  7. Once the count is 11, it's no longer less than or equal to 10, and so the steps of the loop are no longer executed and we move on to the rest of the program.
  8. The last line is then executed, printing out the word "Done."

This is the gist of pseudocode. We just use it to express the algorithm we want to code. Based on this, writing a program in JavaScript, or any other language, is much easier.

Separate Processing and Output

All of these examples clearly demonstrate how we'd solve the problem. But they also do something that can be problematic - they mix output with processing. As programs get larger, this can be a problem.

Look at this program: 

Initialize fullTimeCount to 0
Initialize partTimeCount to 0

prompt for studentAttendanceStatus with "Are you a part-time student?"

IF studentAttendanceStatus is "yes" THEN
  add 1 to partTimeCount
  Display "You are enrolled part time."
ELSE
  add 1 to fullTimeCount
  Display "You are enrolled full time."
ENDIF

This program has two outputs. One in the if part, and another in the else part. If we use a variable, we can separate the display from the branches in our program: 

Initialize fullTimeCount to 0
Initialize partTimeCount to 0
initialize output to ""

prompt for studentAttendanceStatus with "Are you a part-time student?"

IF studentAttendanceStatus is "yes" THEN
  add 1 to partTimeCount
  output = "You are enrolled part time."
ELSE
  add 1 to fullTimeCount
  output =  "You are enrolled full time."
ENDIF

display output

This code is slightly longer, but it's easier to maintain. Should we need to change how we display content, or display additional content later, we can simply add to the output variable. Observe:

This program has lots of displays in it:

DisplayOddAndEven
  initialize count to 1
  WHILE count <= 10 DO
    IF (count modulo 2 == 0) THEN
      display "Even"
    ELSE
      display "Odd"
    ENDIF
    set count = count + 1
  ENDWHILE
  display "Done."
END

Instead of displaying "odd" or "even" in the loop, we could build up a single output string and display it at the end. We can add strings together just like we can numbers, although we call this concatenation.. 

DisplayOddAndEven
  initialize count to 1
  initialize output to ""

  WHILE count <= 10 DO
    IF (count modulo 2 == 0) THEN
      output = output + "Even\n"
    ELSE
      output = output + "Odd\n"
    ENDIF
    set count = count + 1
  ENDWHILE
  output = output + "Done."

  display output
END
