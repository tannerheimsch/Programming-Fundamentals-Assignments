Let's look how we define a variable and then assign data to that variable. This data can then be accessed by our program.

What does data look like?

Numbers:

Here's what the number 7 looks like in the computer:

00000000000000000000000000000111

That's 32 bits in a row, which is maybe a little overwhelming. So, we break up the bits into groups of 8 that we call bytes. (You've probably heard that term, like in a 250GB hard drive. That's 250 billion bytes!)

00000000 00000000 00000000 00000111

For the sake of discussion, we'll not worry about the first 3 bytes since they are all zeros. We'll just look at the last byte.

00000111

Now we'll blow it up and look at what it means. The computer only understands base 2 (binary) arithmetic. The rightmost bit is the ones position, the next one to the left is the twos positions, next is the fours, etc.



Here's what 10 looks like:

00001010

Here's 255:

11111111

And here's 682. But, since it takes more than 8 bits to store that number, we had to add another byte:

00000010 10101010

Characters:

What about letters? Here's what the letter A is in the computer:

01000001

Huh? That looks like a number! Yep, that's what everything is in the computer. That's also the number 65.

So... how does the computer know that it's an A and not 65?

Because we tell it that it's an A. That's what Assignment means.

Assignment

"Assignment" is the process of putting a data value in a variable and telling the computer the value's data type.

What does the term "data type" mean? A data type designates what kind of data will be stored in the variable. For example, will the data be all numbers? If so, it is a number data type...or a "numeric" data type. But, if the value has letters or special characters...a first name, for example...then it's a string data type.

In JavaScript we do this by simply assigning a value to a variable. JavaScript figures out what data type based on how you write the value, using type inference:

  • Numbers: 45, 123.99
  • Strings: "this is a string", "Here's another", "12.99"
  • A number is declared by NOT using quotes.
  • A string is declared with the quotes, even if the value is a number.
  • That's why 12.99 is a number...but, "12.99" is a string!
  • Data types are SUPER important to JavaScript when doing math or comparing values!

There are several ways to do assignment in JavaScript. The simplest is the Simple Assignment Operator.



Let's look at some examples of Simple Assignment Operators

Here, on lines 2 and 3, are examples of a Simple Assignment in a variable declaration statement. By the way, did you notice that Fred has quotes around it? We want JavaScript to know Fred is a string. But, 40000 has NO quotes, because we want JavaScript to see that it's a number.


// Simple Assignment while declaring the variable
var firstName = "Fred";
var income = 40000;

document.write("First Name: " + firstName);
document.write("<br />");
document.write("Annual Income: " + income);

Here's our result...

Here's an example of a Simple Assignment after the variable has been declared. It's on line 4 of our code, but it can happen anywhere in your code after the declaration.


// Simple Assignment after declaring the variable
var petName;

petName = "Fido";

document.write("Pet Name: " + petName);

Here's an example of a Simple Assignment with Math. Look at line 06. On the left side of the equal sign is the variable name...and on the right side of the equal sign is the formula. JavaScript will solve the formula (500 times 345), and place the answer into the variable.


// Simple Assignment with Math
var total;
var countOne = 500;
var countTwo = 345;

total = countOne * countOne;

document.write("The total is: " + total);

Once a variable has a value assigned to it, you can still change it. Here's an example of a Simple Assignment by replacing the value of a variable. Notice that the address variable is assigned a value of "123 Some St." right away on line 2. BUT, on line 7 the address variable is assigned a new value. The new assignment statement always replaces whatever value was assigned to the variable in previous assignment statements. Let me repeat: The old value is gone. Forever.


// Simple Assignment with replacement
var address = "123 Some St.";

document.write("Address: " + address);
document.write("<br />");

address = "456 Another St.";

document.write("New Address: " + address);

Let's look at a Simple Assignment to a new type. JavaScript is what is called a "dynamically typed" language. This means that variables can be any type and you can change their type, because the type of a variable is determined by the data you store in that variable.

See how the variable called notSure is assigned as a String data type in line 02. However...then in line 07, that same variable is assigned a number! JavaScript let's you change the data type simply by assigning a different value to the variable.


// Simple Assignment to a new type
var notSure = "I'm a string for now.";

document.write("Here's notSure: " + notSure);
document.write("\n");

notSure = 25.99;

document.write("New value for notSure: " + notSure);

Many other languages, like Java and C#, are "statically typed" languages. Their variables have to be declared with a specific data type and you can't store any other type of data in that variable. So, the preceding example would result in an error in those languages.

You can also accomplish Simple Assignment from the results of a function. This is very common, and you have already used this method of getting data into a variable!


// Simple Assignment from a function
var userName = prompt("Enter your user name");

document.write("Your user name is: " + userName);

Here's our last example. A Simple Assignment can be done to a variable "in-place", from itself to itself. Sounds strange, doesn't it? But, look at the example in line 08. There is an EXCELLENT reason for that line of code.

  • Did you remember the tricky thing about the prompt function? It is a rule in JavaScript that any variable that is assigned using the prompt function is assigned as a string - even if the value keyed in by the user is a number!
  • The Number function allows JavaScript to look at the value in a String variable, and to do the best it can to convert that value to a number.
  • The Number function is often used with the prompt function. For example, when you clicked the link to get to this page, you were prompted for your name and a price. When you keyed in the "Price", hopefully you keyed in a number. JavaScript sees that number as a string, since it came in from a prompt function. The Number function tells JavaScript to use the "number" value of whatever you keyed in.
  • The number keyed in may not display any differently when you use the Number function. BUT...(a big but)...any math you do with that number may have a VERY different result. More about that later...

// Simple Assignment in place
var priceOne = 15.80;
var priceTwo;
var priceTotal;

priceTwo = prompt("Enter a price.");

priceTwo = Number(priceTwo);

priceTotal = priceOne + priceTwo;

document.write("Total Price is: " + priceTotal);
