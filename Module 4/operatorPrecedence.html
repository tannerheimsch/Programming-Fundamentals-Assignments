Operator Precedence

What is the result of 2+3*4?

It's SUPER important to know the order in which calculations are completed.

Let's look at 2+3*4. If we put it in JavaScript we get:


// Operator precedence
var results = 2 + 3 * 4;
document.write(results);

Is that what you expected? Or did you expect 20?

We get 14 because JavaScript did the (3 * 4) first to get 12 and then added the 2 for a final result of 14 .

If you expected 20 then you did the (2 + 3) first for 5 and then multiplied by 4 to get 20.

Why did JavaScript do it the first way?

Because of Operator Precedence. JavaScript has been told the order to do things when it has been given a group of things to do. That's the "default" order of precendence.

What if we want 20? Then we can change the order that JavaScript will use. We can change the default order by using parentheses. For example:


// Changing precedence with ()
var results = (2 + 3) * 4;
document.write(results);

Here's the output:

JavaScript will do things in parentheses first, always!

Let's look at some examples. First just addition and subtraction


// Addition and subtraction
var test = 2 + 4 + 6 - 10 + 12;
document.write(test);

and the output is:

Still 14, how about this:


// Changing precedence with ()
var test = 2 + (4 + 6) - 10 + 12;

Output:

14 again, can we get a different result?


// Changing precedence with ()
var test = 2 + 4 + (6 - 10) + 12;
document.write(test);

Hmm, maybe not. Let's try one more thing:


// Changing precedence with ()
var test = 2 + 4 + 6 - (10 + 12);
document.write(test);

Whoa! That's different. Could that be right? It sure is, even with just addition and subtraction we have to be careful with operator precedence.

Now let's do some more complicated math.


// Precedence with more operators
var results = 2 + 3 * 4 + 2 / 3 * 6;
document.write(results);

How'd it get 18? Add some parentheses to see if we can see the order that JavaScript used.


// Changing precedence with ()
var results = 2 + (3 * 4) + ((2 / 3) * 6);
document.write(results);

Let's trace through each step

2 + (3 * 4) + ((2 / 3) * 6)
2 + (12) + ((0.666666666666667) * 6)
2 + 12 + (0.666666666666667 * 6)
2 + 12 + (4)
2 + 12 + 4
18

That makes sense now. With something like this we can have lots of different outputs depending on where we put the parentheses. Let's trace through a few and then code them.

2 + 3 * 4 + 2 / 3 * 6
(2 + 3) * (4 + 2) / (3 * 6)
(5) * (6) / (18)
5 * 6 / 18
(5 * 6) / 18

1.6666666666666667


// Changing precedence with ()
var results = (2 + 3) * (4 + 2) / (3 * 6);
document.write(results);

2 + ((12) + 2) / 3 * 6
2 + (12 + 2) / 3 * 6
2 + 14 / 3 * 6
2 + (14 / 3) * 6
2 + (4.66666666666667 * 6)
2 + (28)
2 + 28

// Changing precedence with ()
var results = 2 + ((3 * 4) + 2) / 3 * 6;
document.write(results);

Very different results! So, like type casting, you can avoid confusion by following some simple rules:

Always use parentheses when doing math with more than two parts.
