Operators That Test Relationships

Is the number or string bigger, smaller, or the same? In order to write software that makes decisions, we'll have to do comparisons, and to do that we'll need to test relationships between data. We do that with relational operators.

The Relational Operators

Operator Function
< Less Than test
> Greater Than test
<= Less Than or Equal To test
>= Greater Than or Equal To test

Using the Relational Operators

Relationship testing is done very often in our code, especially in selection or repetition.


// Relational Operators

// if the count is less than 5...
if (count < 5) {
  // do something
}

// while the count is greater than or equal to 10
while (count >= 10) {
  // do something
}

Relational Boolean Expressions

Relational operators make boolean expressions that are either true or false.

Testing Relationships


// Is 10 bigger than 5?
document.write(10 > 5);

Is 10 smaller than 10?


// Is 10 smaller than 10?
document.write(10 < 10);

This means "Is 10 smaller than OR equal to 10", which is true.


// Is 10 smaller than or equal to 10?
document.write(10 <= 10);

What about strings? Can we use this for strings and how would it make any sense?


// Is A smaller than B?
document.write("A" < "B"); // Does this work and make sense?

Yes, it makes sense to compare the relationships of strings. Everything in the computer is a number, even characters like "A". Here's a chart showing the numbers in the computer for characters: What Are Characters?


// Is "F" bigger than "f"?
document.write("Fred" > "fred");

This is why "Fred" with an upper case "F" is smaller than "fred". All upper case letters are smaller than all lower case letters. Thus:


// Is "F" smaller than "f"?
document.write("fred" > "Fred");

So why would we ever compare strings? Well, suppose you had a list of names that you wanted to put things in alphabetical order. JavaScript compares strings character by character. So if you were to compare "Fred" and "Frank", which one would be bigger?


// Is "Fred" bigger than "Frank"?
document.write("Fred" > "Frank");

Less Than (<)

Is the first value smaller than the second?

Numbers. Most of the time we compare numbers with "less than".


// Is 1 less than 10?
document.write(1 < 10);

This one catches people sometimes. What do you think the result will be?


// Is 10 less than 10?
document.write(10 < 10);

Are you surprised by that?

We can also use < with strings. This will compare the numeric values of the strings. 


// Is "b" less than "c"?
document.write("b" < "c");

It works with larger strings, too. It will compare each character from left to right until it finds a difference or it reaches the end. Since the upper case "R" has a lower numeric value than the "r" this returns "true".


// "R" is less than "r".
document.write("smalleR" < "smaller");

Comparing string and numbers? Yes! JavaScript does type conversion for us here too.


// numbers and strings?
document.write(10 < "100");
document.write("\n");
document.write("10" < 100);

There's never a good reason to let JavaScript do this kind of type coercion. Convert strings to numbers before doing comparisons, please. Just because the language will let you be lazy doesn't mean you should.

Greater Than (>)

Is the first value bigger than the second value?

Numbers. We frequently want to compare if some value is bigger than another. For example, is the new salary bigger than the maximum salary for that position? 


// Greater than
var newSalary = 37500.00;
var maximumSalary = 40000.00;
document.write(newSalary > maximumSalary);

Another example showing floating point numbers.


// Greater than with floating point number
document.write(12.555555557 > 12.555555556);

This also works with strings


// Greater than with strings
document.write("ABC" > "abc"); // Why is this false?

Remember how JavaScript compares the characters.

Less Than or Equal To (<=)

Is the first value smaller than or equal to the first value?

Adding an equal sign changes the meaning subtly. This can cause some subtle bugs, too. We now have two questions to ask when we read a statement. First: "Is the first value smaller than the second value?". Second: "Is the first value the same as the second value?". If either of the questions is true then the result will be true.

Here's the same one from the less than section with the added equal sign. It's true now because 10 is equal to 10. A way to read this would be, "Is 10 the same or smaller than 10?". Yes, it is. 


// Less than or equal to
document.write(10 <= 10);

This one has the same result as above.


// Less than or equal to
document.write(1 <= 10);

It works with strings, just like we thought it would.


// Less than or equal to with strings
document.write("b" <= "b");

One situation that can be tricky is when using variables. What if we really meant that when "count" was 10 the result should be false? It is not immediately obvious that we've made a mistake. This becomes an issue when we get to repetition and looping. Keep this in mind for a future unit.


// Less than or equal to with variables
var count = 10;
document.write(count <= 10);

Greater Than or Equal To (>= )

Is the first value bigger than or equal to the second value?

Just like the opposite operator this ones work like > with another question added to the test. With this example, the first question is false but the second question is true. First: "Is 10 greater than 10?". Second: "Is 10 equal to 10?". If either of the questions if true then the results is true.


// Greater than or equal to
document.write(10 >= 10);




// Greater than or equal to with numbers
document.write(12 >= 14);
document.write("\n");
document.write(12 >= 11);

Strings too.


// Greater than or equal to with strings
document.write("fred" >= "Fred");
