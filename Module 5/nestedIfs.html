Nested if/else Statements

What if you want to answer more than one question in a series? Or you want to test a variable for one of many options? We frequently have situations like this in programs. Here's some pseudocode that illustrates a common problem

IF choice1 is "A" THEN
  IF choice2 is "2" THEN
    IF choice3 is "X" THEN
      IF choice4 is "Blue" THEN
        Do a really important task!
      ENDIF           
    ENDIF
  ENDIF
ENDIF

This is the same as using multiple logical AND (&&) expressions, with one important difference: we can use ELSE statements, too.

IF choice1 is "A" THEN
  IF choice2 is "2" THEN
    IF choice3 is "X" THEN
      IF choice4 is "Blue" THEN
        Do a really important task!
      ELSE
        Do a not so important task.
      ENDIF           
    ELSE
      This is if choice3 isn't X. 
      We bypass choice4 altogether.
    ENDIF
  ELSE
    We're here because choice2 is not "2". 
    We'll do something else instead.
  ENDIF
ELSE
    choice1 is not "A"
ENDIF

But, this kind of coding is not that typical. Over time programmers have figured out that this is very fragile and you never really know if it works right. We now have lots of ways to avoid this type of monster. But before we can dig into those advanced methods, you need to understand how nested IF statements work and how they're used.

We'll start with some basic patterns of nesting our logic.

Testing One Variable for Different Values

We frequently need to test one variable for many different values and do something different for each value.

We have a choice variable and the user is asked to make a selection from a list. Doing this with deeply nested if/else statements is ugly! So most languages have better ways. This is usually called the "if, else if, else if, else" pattern among programmers. It looks like this in pseudocode.

IF choice is "A" THEN
  do the A things
ELSE IF choice is "B" THEN
  do the B things
ELSE IF choice is "C" THEN
  do the C things
ELSE IF choice is "D" THEN
  do the D things
ELSE IF choice is "E" THEN
  do the E things
ELSE IF choice is "F" THEN
  do the F things
ELSE IF choice is "G" THEN
  do the G things
ELSE
  do the default thing
ENDIF

That's looks a little better! And here is a JavaScript version of the concept.


// One variable, many choices

var choice = "G";

if (choice === "A") {
  document.write("The choice was A");
} else if (choice === "B") {
  document.write("The choice was B");
} else if (choice === "C") {
  document.write("The choice was C");
} else if (choice === "D") {
  document.write("The choice was D");
} else if (choice === "E") {
  document.write("The choice was E");
} else if (choice === "F") {
  document.write("The choice was F");
} else if (choice === "G") {
  =document.write("The choice was G");
} else {
  document.write("The choice was none of the above");
}

Testing Different Conditions With if/else structures.

Sometimes we have to nest if/else statements as it won't work any other way. We try to limit it and make it concise.


// Nested ifs

var city = "Eau Claire";
var county = "Eau Claire";
var state = "WI";

state = prompt("What State do you live in?","WI");

if (state === "WI") {
  county = prompt("What county do you live in?","Eau Claire");
  if (county === "Eau Claire") {
    city = prompt("What city in Eau Claire county do you live in?","Altoona");
    if (city === "Eau Claire") {
      document.write("Eau Claire is a nice place");
    } else {
      document.write("There are other cities in Eau Claire County?");
    }
  } 
  else {
    city = prompt("What city do you live in?", "Osseo");
    if (city === "Osseo") {
      document.write("Too many Norwegians in Osseo.");
      } else {
      document.write("I don't know much about that city.");
    }
  }
} 
else {
  document.write("Goodbye!");

Range Testing

We frequently need to test to see if a value is within a range. The technique to solve this combines nested if/else with multiple booleans.

Pick a number between 1 and 10


// Range Testing

var number;

//number = prompt("Pick a number between 1 and 10");

number = Number(number);

if (number >= 1 && number <= 10) {
  document.write("You got it right: " + number);
} else {
  document.write("Your entry was out of range.");
}

Using this technique we can pick from multiple ranges.


// Multiple ranges

var number = 0;

number =  prompt("Pick a number between 1 and 50",25);

number = Number(number);

if (number < 0) {
  document.write("That number is negative, I don't do negative.");
} else if (number >= 0 && number <= 10) {
  document.write(number + " is between 1 and 10");
} else if (number > 10 && number <= 20) {
  document.write(number + " is between 10 and 20");
} else if (number > 20 && number <= 30) {
  document.write(number + " is between 20 and 30");
} else if (number > 40 && number <= 50) {
  document.write(number + " is between 40 and 50");
} else {
  document.write(number + " is bigger than 50.");
}
