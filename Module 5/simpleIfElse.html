Unit 5: Making Decisions

Simple if/else Statements

We use IF/ELSE statements when we need to "branch" our code, or make an "either-or" type of decision. IF/ELSE statements are for those situations where you have to make a binary choice. For example, you're either able to vote or you're not. You're either an admin or you're not.

While we could use two simple IF statements to do the same thing, it's a Very Bad Idea to do that, because it would make the computer check the same value more than once. So, for "either or" situations, we have:

The if/else Statement

Pseudocode

IF some condition THEN
  turn left
ELSE
  turn right
ENDIF

In JavaScript, this is how your if/else code should look.


// Simple if/else

if (boolean expression) {
  // do something
} else {
  // do something else
}

How does this work?

If the boolean expressions is true then the code after the if is run. If it is false then the code after the else is run.

Examples

Here's the above example in running code. 


// if Example, false
var count = 4;

if (count === 5) {
  document.write("yes");
} else {
  document.write("no");
}

Here's one that's true


// if Example, true
var price = 14.99;

if (price < 15.00) {
  document.write("I'll buy it!");
} else {
  document.write("That's too much for me.");
}

Where it gets interesting is when we won't know which way it will go until the program is running.


// Which way will it go?

function WhichWay() {
  // Variable Declarations
  var output = "";
  var priceOne;
  var priceTwo;
  var total;
  var subtotal;

  // Get user input and convert to numbers.

  priceOne = prompt("Enter 1st price.");
  priceTwo = prompt("Enter 2nd price.");

  //Convert input to numbers
  priceOne = Number(priceOne);
  priceTwo = Number(priceTwo);

  // Calculate total and store it so we
  // can use is more than once
  subTotal = priceOne + priceTwo;

  // Which discount should we apply?
  if (subTotal > 24.99) {
    total = subTotal - (subTotal * 0.1);
  } else {
    total = subTotal - (subTotal * 0.05);
  }

  // Display output differently depending on the discount
  // Here's where we use the subTotal again. Isn't it handy
  // that we stored it?
  output = "Your total with a ";
  if (subTotal > 24.99) {
    output += "BIG ";
  } else {
    output +="small ";
  }

  output += "discount on a purchase of $" + subTotal +
            " is $" + total;

  document.write(output);
}

// run program - uncomment to run.

Full if/else Example

IF / ELSE statements aren't just for controlling input and output. They're for "control flow" and that's really important when you write complex programs. When we "branch" our code, we can literally go down two completely different paths. We can, for example, check the value a user gives us, and only run the rest of our program if the value they give us is valid.
