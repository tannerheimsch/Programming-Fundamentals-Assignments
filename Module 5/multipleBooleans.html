If Statement With Multiple Boolean Expressions

This OR That, This AND That

Sometimes we want to have multiple conditions because one isn't enough.

  • "If I were taller or that shelf was lower, I could reach that can of soup."
  • "Could I have 20 ones or a ten and 10 ones or even 2 fives and 10 ones?
  • "You have to wear a shirt and shoes to enter this premise!"
  • "If you are older than 5 and younger than 12 then the cost is $1"

The Logical OR (||)

In JavaScript, as in many languages, the logical OR is ||. That's two vertical bars. That key is in different places depending on the keyboard. Where is it on yours? You use it like this:

In pseudocode we use the word OR in capital letters.

IF number > 10 OR number < 0 THEN
  display error message
ENDIF

In JavaScript we use two vertical bars, ||.


// Logical OR

var number = 25;

if (number > 10 || number < 1) {
  alert("The number is not between 1 and 10!")
}
  • Either one of the expressions can be true and the whole thing is true!
  • Notice that you have to repeat the variable after the ||. This is wrong:

// Wrong!
if (number > 10 || < 1) { //<-- Wrong, missing variable name!
  • This is correct:

// Right!

if (number > 10 || number < 1) { //<-- You always have to have
                                       // a complete boolean expression
                                       // on each side of the ||

The Logical AND (&&)

In pseudocode we use the AND in uppercase letters.

IF it's the weekend AND I'm not tired THEN
  We're going to the park
ENDIF

IF numberOne > 25 AND code is 5 THEN
  Award bonus
ENDIF

In JavaScript the logical AND is &&, two ampersands. Here's how we use it:


// Logical AND

var code = 5;
var numberOne = 30;

if (numberOne > 25 && code === 5) {
  document.write("Bonus time!");
} else {
  document.write("No bonus this year.");
}

Both of the boolean expression has to be true. If either one is false then the whole thing is false.

Examples


// Logical AND example

function LogicalAndExample() {
  var count;

  count = Number(prompt("Enter a number between 1 and 10"));

  if (count > 0 && count <= 10) {
    document.write("The number is in the correct range");
  }

}

  • Here's an if/else

// Logical AND example with if/else
function LogicalAndExampleWithIfElse() {

  var code;
  var output;
  var size;

  size = prompt("Enter a size in feet");
  code = prompt("Enter a code: A, B, or C");

  size = Number(size);

  if (size > 0 && code === "B") {
    output = "That is just right!";
  } else {
    output = "That's not it, please try again.";
  }

  document.write(output);
}

Short Circuiting

Look at this code:


(2 === 2) || (foo);

This returns true. We didn't declare the variable foo, but because the first part of this condition is true, JavaScript "short-circuits" and returns true, never even trying to read the second part. It only reads the second part if the first part is false.

 


(2 === 3 ) || (foo);

Since foo isn't defined, JavaScript gives us an error and the program stops running.

The important thing to remember is that if the first part is true, the rest is ignored. If the first part is not true, then the second part happens.

(Notice I don't say "false", I say "not true.")

Why would we do this?

We can use booleans to set default values. For example, instead of this: 


var defaultState;
if(userValue === ""){
  defaultState = "WI";
}

We can do this:


var defaultState = userValue || "WI";

This is an example of "short circuiting" and it's really important when understanding how booleans in JavaScript work. Not every language has short-circuiting, but many do. We won't do any short-circuiting in this class, but you should be aware of its existence.
