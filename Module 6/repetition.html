Repetition Programming

Repetition is the last of three main computer concepts that we're learning. Sequence, Selection, and now Repetition.

Repetition means Looping

In programming we always use a loop when we want our program to repeat something.

We have three main looping structures that we're going to learn in this course.

  • Leading Decision Loop
  • Trailing Decision Loop
  • Counted Loop

Leading Decision Loop

Test the condition - and make the decision FIRST.

We saw this pseudocode earlier in the semester

WHILE condition p is true
  statement block
ENDWHILE

What we didn't emphasize is that "condition p is true" is checked first, before the code inside the loop is run. If the condition is true then the code is run.

After all the statements in the loop are run then "condition p is true" is tested again. If it is still true then the code in the loop is run again.

Then the condition is checked again and we continue that process (called a loop) until condition p is NOT true.

Interesting...that if the condition is NEVER true - the statements are NEVER run!

In JavaScript the syntax would look like this:


// JavaScript "while" loop
while (condition === true) {
  // statement block here
}

Here's an actual code example:

// JavaScript "while" loop
// Declare variables.

var city;
var moreCities = 'no';

// prompt for and display cities until there are no more

moreCities = prompt("Are there more cities?");

while (moreCities === 'yes') {  //begin loop
  city = prompt("Please enter a WI city: ");
  document.write("Printing WI cities: " + city);
  document.write("\n");
  moreCities = prompt("Are there more cities?");
}

//display final message after loop ends
document.write("That's all of the cities in WI.");    
document.write("\n");

Trailing Decision Loop

Test the condition - and make the decision AFTER the fact. .

Here's the pseudocode for this concept:

DO 
  statement block
WHILE condition p is true

With this loop the statement block is run before the test is done. This allows us to guarantee that the statement will run at least one time.

Interesting...that means the code statements are ALWAYS run at least once - even if the condition is never true!

After the statement block is run the test will be run. If "condition p is true" then the statement block will be run.

Then the test is done again.

Then the statement block is run again.

This continues until "condition p is true" is now false.

This is the JavaScript syntax:

// JavaScript "do while" loop
do {
  // statement block here
} while (condition === true);

Notice the semicolon at the end of the code. Because this statement doesn't end with a } we need the semicolon.

And, here is an actual JavaScript code example:

// JavaScript "do while" loop
// Declare variables
var city;
var moreCities = 'yes';

//Loop as long as there are more cities
do {
    city = prompt("Please enter a WI city: ");
    document.write("Printing WI cities: " + City + "\n");
    moreCities = prompt("Are there more cities?");
} while (moreCities === 'yes'); 

//Display message after loop ends
document.write("That's all of the cities in WI.\n");

Counted Loops

Repeat the statement block a specific number of times.

The concept of counted looping is simple, you do something a specified number of times. There are different ways to specify the number. Here's the first way in pseudocode:

set counter to zero
WHILE counter < 10
  statement block
  add 1 to counter
ENDWHILE

We create a variable and initialize it to 0.

We start the loop by testing our variable against the number of times we want to loop. In this case we want to loop 10 times.

Then we run the statement block, this will be our working code.

At the end of the loop we have to add 1 to our "counter" variable. If this step is forgotten our loop will run forever.

There is more than one way to code this in JavaScript. Here's the syntax that matches the pseudocode.


// Looping a fixed number of times

var counter = 0;

while (counter < 10) {
  // statement block here        
  // add 1 to counter at the end of the loop
}

Here's an actual code example:

// Looping a fixed number of times
// Declare variable
var myCounter = 0;

// Loop 10 times and display the counter's value  
  
while (myCounter < 10) {
  document.write("The value of the counter is: " + myCounter + "\n");     
  myCounter++;  // add 1 to counter at the end of the loop
}

while loops are great, but if you're not careful you'll create the dreaded Infinite Loop! When using the while loop like this, you can't forget to have a condition that will eventually not be true. If you forget to increment your counter variable, you'll loop forever.

For Loops

while loops are better suited for loops where we don't know how many times we need to repeat. For a counted loop, we usually use a for loop. Here's an actual code example:


// JavaScript counted loop using the "for" loop

for (var myCounter = 0; myCounter < 10; myCounter++) {
  document.write("The value of the counter is: " + myCounter + "\n");
}

The for loop does exactly the same thing as the previous while loop. However, managing the loop counter is built in which makes it easier to use. For example, you can't forget to have a counter variable or it won't run. And, you can't forget to increment the counter either!

Here are the basics of how a for loop works.

  • The for statement has three parts in between the parentheses, initialization, test, increment.
    • (myCounter = 0; MyCounter < 10; myCounter++) - First the myCounter variable is set to zero. This initialization happens only once before the loop starts.
    • (myCounter = 0; myCounter < 10; myCounter++) - This middle part is the test, it's just the same as inside a while loop's parentheses. It's the last thing checked before the loop starts again.
    • (myCounter = 0; myCounter < 10; myCounter++) - At the end is the increment step, which is run after the loop statements are run.

The for loop is more concise and is preferred when we know how many times a loop should be run.

However, it's also easy to goof up. You need to be mindful that you declare your variable and give it the correct initial value. You need to be mindful that your stop condition is correct.

You need to watch your semicolons too. Look at this code:


for (var myCounter = 0; myCounter < 10; myCounter++);
{
  document.write("The value of the counter is: " + myCounter + "\n");
}

Can you guess why that only prints one result?

It turns out that for loops don't have to have a body. The statements in the for loop's declaration still run even if there's no declared body. So the myCounter variable is initialized to 0 at the beginning, and then it still gets incremented. Since myCounter is incremented at the end of each iteration, the value of myCounter will be one more than the number of times it loops.

Also, be sure you keep your for loops consistent with your program's requirements. If the program says to count from 1 to 5, be careful. Look at this code:


for (var myCounter = 0; myCounter < 5; myCounter++) {
  document.write("The value of the counter is: " + myCounter + "\n");
}

This loops 5 times, but the variable is initialized to 0, so it only prints out 0 to 4. Since the problem calls for 1 to 5, just do that in the code.


for (var myCounter = 1; myCounter <= 5; myCounter++) {
  document.write("The value of the counter is: " + myCounter + "\n");
}

See? The loop uses 1 and 5. We use <= because that makes sure we include 5. If we simply used < 5, we'd only get 1 to 4 printed. We call these "off by one errors"

"There are two hard problems in software development: Naming things, expiring caches, and off-by-one errors."

You can keep the off-by-one errors at bay by working hard to keep the program's code in sync with the problem you're solving. If it asks for 1 to 5, use 1 to 5. Don't use 0 < 5 or 1 < 6. Use logic to express exactly what has to happen while keeping the numbers consistent.

Infinite for Loops

for loops can be infinite too. 


for(;;){
 // do this forever.
}

Leaving all of the statements off of a for loop sets it to run infinitely. And this is actually the preferred method for doing an infinite loop in some languages. (Yes, there are reasons to write infinite loops. But those are intentional, not accidental.
