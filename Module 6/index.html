What can a computer do better and faster than a human? It can repeat the same action over and over (and over and over...) - without making a mistake or getting bored!

In programming, we refer to that concept as "repeat programming", "Looping", or "Iteration." Using certain types of code statements, we can instruct the computer to complete an activity, and then repeat that activity over and over. Our program code also tells the computer when to stop repeating that process.

For example, imagine a computer program that determines how much each employee gets paid each week. For each employee the program must read the number of hours that employee worked and the employee's hourly wage from a database, then multiply the two, then update another table in the database. That sounds pretty easy! We might be able to do that in just 5 or 6 lines of code!

But...what if the company has 200 employees? Five or 6 lines of code multiplied by 200...that would be a LOT of lines code - all doing the same thing. What a waste of time! Code you write is code you have to maintain, too!

Using a "repeat programming construct" in our code we will tell the program to reuse or "loop through" that 5 or 6 lines of code for each employee. And, by selecting the appropriate type of loop, we'll tell the computer when to STOP repeating the code, too. For example, we might include code to tell the computer to:

  • Repeat those lines of code UNTIL something happens.
  • OR...repeat those lines of code WHILE some condition is TRUE.
  • OR...even this...repeat those lines of code a certain number of times.
